  • The main disadvantage is that all the paragraphs are applied with same style. To make a specific part of HTML styled a perticular way, (say we want the middle paragraph to be yellow) "id Selector" was needed.

    • id Selector

  • Here we specify the tag we want to stylize with a unique id.
  • We access that tag using the id we gave in css document using (Hashtag-#).
  • An id is a unique identity of a tag, so no two tags can have same id.
  • Also a single tag cannot have more than one id.

    • Example : we used syntax - #id{ CSS code }

  • Now suppose we have to make Both the heading and the paragraph-1 and paragraph-3 aqua, dowe have to wtrite the same code 3 times mentioning their "id"? isnt it redundant?
  • Thats why "class Selector" came into use.

    • class Selector

  • We can group together different tags under same class using 'class' attribute. The benifit of this is that we can apply same style to all of them simultaneously without the need to write code for them individually.
  • We access that tag using the class we defined for it in css document using (dot .)

    • Example : we used syntax - .class{ CSS code }

Priority Of different Selectors

  • As we already know that the priority of inline CSS is the highest and between internal CSS and external CSS the one who came later takes the priority. [ If the style is done on same tag ]
  • The above rule is valid only for individual selectors.
    • Meaning :
  • If we modify a pararaph using multiple 'id - selectors' then the one who comes at the bottom will be implemented. (same for others if used individually)


  • If we use multiple selectors on the sme tag, priority is set as : "id-Selector" > "class-Selector" > "element-Selector", regardless of their position in the code.
