Introduction

  • CSS = Cascading Style Sheets
  • It determines the design of an HTML page and how it will looklike in different media and devices.

Syntax :

Selector

  • This specifies the part where the css is going to be applied.
  • There are 3 types of base selectors in HTML :
    1. Element Selector
    2. id Selector
    3. class Selector

  • Each Selectors in Detail :
    1. Element Selector :
      • It directly specifies the part by the help of HTML tags.
      • The problem with Element Selector is that : If we directly write a genralized tag like : <p> (paragraph), <head> (head) etc.. then it will apply the style on all present (paragraph/head) tag of the html file.
      • This limits us the ability to customize each and every part of HTML by our own wish.

    2. id Selector :
      • It uses an attribute called 'id' .
      • 'id' is used to give a tag a unique identification.
      • We can access an 'id' in css as selector using # (hashtag). in selector place just write '#id_name' { css code }
      • It is a developer convention to use only one unique id for each tag, it is not advised to have multiple tags with same id.
      • Also a single tag cannot have multiple ids.

    3. class Selector :
      • It uses an attribute called 'class' .
      • 'class' attribute is also used to give a tag a unique identification just like 'id' attribute.
      • But unlike 'id' we can have more than one tag with same class, just to stack them together as same unit.
      • We can access an 'class' in css as selector using . (dot), in selector place just write '.class_name' { css code }
      • Unlike 'id' a tag can have multiple classes.


Property Name & Value

  • They make up the syntax of different style operations.
  • Property Name is the style operation.
  • Value is the way we customize the Property Name.
  • It is important to finish the statement with a semicolon.

Ways of applying CSS in an HTML file

There are 3 ways of implementing CSS in HTML file :

  1. External : We can create an external CSS file and link it to our current HTML file using <link> tag in head.
  2. Internal : We can internally include CSS under <style> tag in head
  3. Inline : we can directly do css in a tag by including 'style' attribute in that tag, we dont have to use selector for it as the affected tag is the current one.

Priority of applying CSS Externally, Internally or in Inline

  • A single HTML file can contain multiple CSS instances while all beloning different modes of application [External, Internal,Inline]
  • It is not even mandatory to write CSS in (head-tag) it can also be written in body or footer and will work fine, but as a convention we use it in head.


  • Priority wise , Inline has the highest priority. If External, Internal & Inline CSS all are affecting a tag at the same time, the CSS in Inline mode will be applied always.
  • In between External and Internal, The priority shifts to who got defined later.
  • If both External and Internal CSS are changing a perticular tag in an HTML file , then it depends on which tag got declared later in the code. If the External Declaration comes later to Internal Declaration then The External one will be applied and vice versa. (considering no Inline Declaration is there)
