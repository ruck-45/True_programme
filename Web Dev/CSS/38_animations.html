Animations

  • We can apply animations apart from Javascript, through CSS.

  • @keyframes (animation-name) { Frames }
    • It is used to define each frames/stages of the animation.
    • if we have just 2 stages in animation, we can write them as-

      • @keyframes change_color { from{ background-color: red; } to{ background-color: yellow; } }

    • if we have more than two stages we can precisely define the frames using percentage.
    • the percentage is the fraction of the "animation-duration" mentioned below at which the transition will occure.
    • Example:

      • @keyframes change_color { 0%{ background-color: red; } 25%{ background-color: yellow; } 50%{ background-color: red; } 75%{ background-color: yellow; } 100%{ background-color: red; } }



  • animation-name: (name) ;
    • we use it to apply the animation we defined using "@keyframes" to a specific element.


  • animation-duration: (time in s or ms) ;
    • defines the time duration for one cycle of the animation.
    • The percentage given in "@keyframes" are fractions of this time duration specified.


  • animation-delay: (time in s or ms) ;
    • defines the time after which the animation will start.


  • animation-iteration-count: (number of iterations) ;
    • defines the number of times the animation will loop itself before coming to a halt.
    • We can write- animation-iteration-count: infinite; to make the animation continue looping forever.


  • animation-timing-funtion: (different modes) ;
    • Just like "transition-timing-function" , "animation-timing-function" also has same set of modes like - ease, ease-in, ease-in-out, ease-out ... etc.
    • It specifies the nature of transition in our animation.


  • animation-direction: (different modes) ;
    • It has different modes to control the direction of the animation.
    • Some modes are :
      • reverse - reverses the anmation direction, starts the animation at the bottom frame.
      • alternate - starts the animation normally and then alters the direction in every iteration.
      • alteranate-reverse - alters the direction in every iteration just like "alternate", but starts in reverse direction first
