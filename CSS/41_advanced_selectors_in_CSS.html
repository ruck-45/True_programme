All Selectors In CSS

  • Element / Type-Selector:
    • selects the base tag.
    • all elements belonging to the selected ta in HTML file is affected.

    Example :

    h1{ CSS code; } p{ CSS code; } div{ CSS code; }


  • id-Selector:
    • unique id is given to tags.
    • id is selected through - "#" (hash).
    • By convention no two elements should have same id in an HTML.

    Example :

    #id{ CSS code; }


  • class-Selector:
    • tags are associated with one or more class.
    • class is selected through - "." (dot).
    • By convention elements could belong to two or more class in an HTML [separated by space] .

    Example :

    .class{ CSS code; }


  • psudo class & psudo element-Selector:
    • they can be used to select specific portion of tags.
    • useful psudo class : first-child, last-child, nth-child etc...
    • useful psudo elements : first-letter, first-line, selection, before, after etc...

    Example :

    a:first-child{ CSS code; } div:nth-child(4){ CSS code; } p::first-line{ CSS code; }


  • Attribute-Selector:
    • they select the tags who have the attribute mentioned here.
    • it can be used in case of tags who have various versions depending upon their attribute, like: "input-tag"

    Example :

    input[type="text"]{ CSS code; } input[type="button"]{ CSS code; } a[href="https://www.google.com"]{ CSS code; }

    • in first example : that input-tag will be selected which has the type attribute as "text".
    • similarly in second example : that input-tag will be selected which has the type attribute as "button".
    • in the third example, the anchor tag whch has its address set to google search engine will be selected.

  • Universal-Selector:
    • "*" (star) is used to select each and every tag in HTML file.
    • It is not much used except the debu phase.
    • We mainly use it to give every element a border to see element collisions and spacing issues if any.

    Example :

    *{ CSS code; }


  • Combinator-Selector:
    • There are many different types of combinator selectors.
    • They are :

      • Descendant combinatory selector: (child at any stage)
        • the selectors are written space separated.
        • every selector to the right represents the child of the selector in the left at any stage. (the child can be immidiate or more levels down)

          • Example :

          #id_01 p{ CSS code; }


      • Child combinatory selector: (Direct Child)
        • the selectors are written separated by ">".
        • every selector to the right represents the immidiate child (just one level below) of the selector in the left.

          • Example :

          #id_01>p{ CSS code; }


      • Adjacent combinatory selector: (Immidiate Sibling)
        • the selectors are written separated by "+".
        • every selector to the right represents the immidiate sibling (on the same level and just neighbouring) of the selector in the left.

          • Example :

          #id_01+p{ CSS code; }


      • General Sibling combinatory selector: (General Sibling)
        • the selectors are written separated by "~".
        • every selector to the right represents the general sibling (on the same level but may or may not be neighbours) of the selector in the left.

          • Example :

          #id_01~p{ CSS code; }



  • Add-Selector:
    • multiple selectors can be given same CSS code using this.
    • selectors are separated by "," (commas)

    Example :

    .class_01,a,p{ CSS code; }
